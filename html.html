O que é HTML?


HTML (HiperText Markup Language) é uma linguagem de marcação usada para construir páginas web.

Marcação?


As marcações servem para estruturar as páginas através de cabeçalho, corpo e rodapé. Além de incluir parágrafos, imagens, listas e formulários.

Como utilizar o HTML?


Os documentos HTML são interpretados pelo navegador, e podem ser criados através de diversas ferramentas. Indo das mais simples até as sofisticadas.

Evolução do HTML


HTML

Criada por Berners-Lee para que pudesse compartilhar pesquisas com o seu grupo.

Evolução do HTML


HTML 2.0

Atributos para definir tamanhos, tipos e cores de letras, cores de fundo, textura, etc.

Evolução do HTML


HTML 3.0

Atributo class, estilização, tabelas, notas de rodapé e formulários.

Evolução do HTML


HTML 3.2

Incorporou os elementos table e applets.

Evolução do HTML


HTML 4

Endossada pela W3C como uma recomendação oficial.

Evolução do HTML


HTML 4.01

Possui 3 variantes: Strict, Transitional e Frameset.

Evolução do HTML


HTML 5

Versão atual

HTML 5


Surgiu a partir da cooperação entre a World Wide Web Consortium (W3C) e a Web Hypertext Application Technology Working Group (WHATWG);

HTML 5


O acordo definiu algumas regras:

  • Reduzir o uso de plugins externos (Flash, por exemplo);
  • Melhor tratamento de erro;
  • HTML 5 deve ser independente do dispositivo;
  • HTML 3.2: Incorporou os elementos table e applets.
  • O processo de desenvolvimento deve ser visível ao público.

HTML 5


Novidades

  • O elemento canvas para desenho em 2D;
  • Os elementos video e audio para reprodução de mídia;
  • Novos elementos específicos de conteúdo, como article, footer, header, nav e section;
  • Novos controles de formulário: calendário, data, hora, email, url e pesquisa.

O que são Tags?


Tags são rótulos (etiquetas) usados para informar ao browser como ele deve apresentar o website;

Como utilizar?


As marcações servem para estruturar as páginas através de cabeçalho, corpo e rodapé. Além de incluir parágrafos, imagens, listas e formulários.

Como utilizar o HTML?


As tags são conhecidas por serem envolvidas pelos sinais < > e < / > ;

Calma, não entendi muito bem...


O formato genérico de uma tag é : < > conteúdo < / > , ou seja, todo conteúdo necessário para uma página web ficará dentro de uma tag específica.


A / indica que a tag está finalizando a marcação de um texto.

Estrutura básica do HTML 5


< !DOCTYPE html >
< html >
   < head >
      < title > Aqui fica o título da minha página < / title >
   < / head >
   < body >
      Aqui fica o conteúdo da minha primeira página HTML
   < / body >
< / html >
  • < !DOCTYPE > deve ser a primeira instrução de um documento HTML, antes mesmo da < html >;
  • A declaração não é uma tag, mas sim uma instrução que sinaliza ao navegador qual a versão está sendo utilizada naquele documento;

Exemplo 


< !DOCTYPE html >
< html >
   < head >
      < title > Tag H < / title >
   < / head >
   < body >
      < !-- Testando as tags de comentário e de cabeçalho -->
      < h1 style ="color:blue;text-align:center"> Cabeçalho de Nível 1 < / h1 >
      < h2 style ="text-decoration:underline;text-align:right; font-style:italic"> Cabeçalho de Nível  2 < / h2>

Hipertexto - Tag < a >


  • Um hipertexto (ou link) é uma forma não linear de apresentar a informação textual;

  • Outros conjuntos de informação são agregados (textos, palavras, imagens ou sons) cujo acesso se dá através de referências específicas denominadas links.

  • Em HTML, para criar um link utiliza-se a tag < a > e o seu principal atributo é o href que indica o destino.

  • O atributo target indica como o destino será exibido;

  • Em < body >, podemos utilizar os atributos link, alink e vlink para especificar a cor de cada um dos links de acordo com o seu status.

Exemplo 


< !DOCTYPE html >
< html >
   < head >
      < title > Hiperlinks < / title >
   < / head >
   < body link="blue" alink= "red" vlink="green"> Clique < a href="https://sites.google.com/site/fabioomedeiros/autoria-web" target="_blank"> aqui < / a > para abrir o site da disciplina em < b > outra página < / b > < br > < br > Já < a href="http://www.ifrn.edu.br"> neste link < / a >, o portal do IFRN será aberto < b > nesta mesma página < / b >
   < / body >
< / html >

  • Link: Link não visitado.
  • Alink: Link ativo.
  • Vlink: Link visitado.

Caminhos

  • Um dos erros frequentes de pessoas que iniciam a construção de sites é a definição errada dos caminhos dos arquivos;

  • Ao inserir um elemento (imagem, som, etc) é necessário definir claramente onde o recurso está para que o browser o encontre;

  • Assim, existem 2 formas para definir o caminho dos arquivos: Caminho absoluto e caminho relativo.

Caminho absoluto

Caminho que o navegador precisa seguir para encontrar o arquivo, definido desde o início. Por exemplo: 



http://www.medeiros.com/sistemas/sigai/cadastro.html

  • O exemplo indica o caminho absoluto do arquivo cadastro.html, localizado na pasta sigai, que está na pasta sistemas, localizada no domínio medeiros;
  • Em geral, os caminhos absolutos são usados para ‘”linkar” arquivos que estão fora do domínio de nosso site.

Caminho relativo

  • Tomam o próprio documento HTML como referência para encontrar outros elementos, desde a página até o arquivo;

  • Neste caso, não precisa escrever todo o caminho para chegar até ele (caminho absoluto), mas sim como chegar até ele;

  • Imaginemos a seguinte estrutura do site:

Caminho relativo 


http://www.medeiros.com
		index.html
		contato.html
		sistemas
		      sigai
		        index.jsp
		        cadastro.html
		        consulta.html  
		      sicon
		        index.php
		imagens
		      background.jpg
		      logo.jpg
		      foto.jpg

    Tags < table > e complementares

    • As tabelas são bastante usadas pelo designer de uma home-page. Com elas é possível fazer alinhamentos que, dificilmente, seriam possíveis com simples comandos;

    • A tag < th > adiciona cabeçalhos: Table Header;
    • A tag < tr > inicia uma linha: Table Row;
    • A tag < td > inicia uma coluna: Table Data;
    • A tag < caption > define uma legenda;
    • As tags < thead >, < tfoot > e < tbody > são usadas para delimitar o cabeçalho, o rodapé e o corpo de uma tabela, respectivamente.

             < / tr >

    O que são Formulários?


    Os formulários são estruturas que permitem aos usuários submeterem dados a uma página;

    Qual a utilidade?


    São bastantes utilizados para realizar buscas e/ou introduzir dados pessoais;

    Como usar?


    Os formulários são delimitados pelas tag < form > .

    Atibutos

    Action - Define a ação que será realizada sobre o formulário. São duas:

  • formulário enviado a um endereço de e-mail: 
    • 
< body >
   < form action = "mailto:endereco@correio.com" ... > 
< / body >
  • formulário enviado a um programa/script que processa o conteúdo: 
    • 
< body >
   < form action = "enderecoArquivo" ... > 
< / body >

    Atibutos

    Method – especifica a forma como o formulário será enviado. São dois:

  • get – as variáveis são passadas pela URL para a página destino: 
    • 
< form action = "mailto:ze@yahoo.com.br" method = "get" > ... < / form >
  • post – as variáveis são passadas de uma forma transparente para o usuário. Por conta disso, POST é o método mais recomendado: 
    • 
< form action = "http://www.ifrn.edu.br" method = "post" > ... < / form >

    Métodos de envio: GET e POST

  • Ambos cumprem a mesma tarefa (obter os dados de um formulário e enviá-los a um servidor web), mas de maneiras diferentes;
  • GET empacota as variáveis do formulário, porém as variáveis são anexadas no final da URL antes de enviá-las para o servidor: 
    • 
http://www.procopio.com.br/sucesso.html?txtLogin=ifrn&txtSenha=123456
  • POST também empacota as variáveis, mas faz o envio de forma oculta para o servidor: 
    • 
http://www.procopio.com.br/sucesso.html

    Métodos de envio: GET e POST

    GET

  • Não deve ser usado para informações case sensitive;
  • Há limitações de tamanho.

    • POST

  • Sem limitações de tamanho;
  • Não exibe os campos submetidos.

    • [W3C, 2017]

    Tag < label >

  • Define um rótulo (uma etiqueta) para um elemento input; 
    • 
< form action = "confirmacao.html" method="post" name= "cadastro" >
   < table >	
     < tr >
       < td > < label for= "txtLogin" > Login: < / label > < / td >
     < /tr >
   < / table >	
< / form >

    Observação: Usaremos a tag table para auxiliar o alinhamento dos formulários.

     < / tr >

       < td > < label for= "txtSenha"> Senha: < / label > < / td >
       < td > < input type="password" name="txtSenha" value="" autofocus> < / td >
     < / tr >      
< / table >

    < / tr >
< / table >

    Fieldset e Legend

  • Os elementos < fieldset > e < legend > criam uma borda (que contorna o formulário) e uma legenda, respectivamente;
  • A tag < form > deve ficar dentro de < fieldset > : 
    • 
< fieldset >
   < legend > Cadastro < / legend >
   < form action = "confirmacao.html" method="post" name= "cadastro" >
            . . .
   < / form >
< / fieldset >

    Atributos Pattern e Placeholder

  • O atributo pattern permite definir expressões regulares sem usar JavaScript. Já o placeholder um pequeno texto sobre o valor que deve ser atribuído a um campo:

    . . .
